Understanding SQL Injection: From Basics to Advanced

Published: January 15, 2025 • 10 min read

Introduction

SQL Injection remains one of the most critical web application vulnerabilities, consistently appearing in the OWASP Top 10. Despite being well-known, many applications still fall victim to this attack vector. In this comprehensive guide, we'll explore SQL injection from the ground up, covering detection, exploitation, and prevention strategies.

What is SQL Injection?

SQL Injection (SQLi) is a code injection technique that exploits security vulnerabilities in an application's database layer. Attackers can manipulate SQL queries by inserting malicious SQL code into input fields, potentially gaining unauthorized access to sensitive data, modifying database contents, or even executing administrative operations.

Types of SQL Injection

  • In-band SQLi - Results displayed directly in the application
  • Blind SQLi - No direct output, relies on application behavior
  • Out-of-band SQLi - Data exfiltration through different channels

Basic Example

Consider a vulnerable login form with this SQL query:

SELECT * FROM users WHERE username = 'admin' AND password = 'password123'

An attacker could input the following as the username:

admin' OR '1'='1' --

This transforms the query into:

SELECT * FROM users WHERE username = 'admin' OR '1'='1' --' AND password = 'password123'

The -- comments out the rest of the query, and '1'='1' is always true, bypassing authentication.

Detection Techniques

Before exploitation, you need to identify vulnerable parameters:

1. Single Quote Test

Insert a single quote ' into input fields. If you receive a SQL error, the parameter is likely vulnerable.

2. Boolean-Based Detection


' OR 1=1 --
' OR 1=2 --

If different responses occur, the application is vulnerable to boolean-based blind SQLi.

3. Time-Based Detection


' OR SLEEP(5) --
' OR pg_sleep(5) --

If the application delays, time-based blind SQLi is possible.

Exploitation

Once a vulnerability is confirmed, you can extract data systematically:

Database Enumeration


' UNION SELECT NULL, database(), NULL --
' UNION SELECT NULL, table_name, NULL FROM information_schema.tables --
' UNION SELECT NULL, column_name, NULL FROM information_schema.columns WHERE table_name='users' --

Prevention

Protecting against SQL injection requires multiple layers of defense:

1. Prepared Statements


// Python example
cursor.execute("SELECT * FROM users WHERE username = ? AND password = ?", (username, password))

2. Input Validation

Validate and sanitize all user inputs. Use whitelists, not blacklists.

3. Least Privilege

Database accounts should have minimum required permissions.

4. Web Application Firewall

Implement a WAF to detect and block SQL injection attempts.

Conclusion

SQL Injection continues to be a prevalent vulnerability due to poor coding practices and lack of security awareness. By understanding how these attacks work and implementing proper defenses, developers can significantly reduce the risk of SQL injection vulnerabilities in their applications.

Remember: security is not a one-time implementation but an ongoing process. Regular security audits, code reviews, and penetration testing are essential to maintaining a secure application.
